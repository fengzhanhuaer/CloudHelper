# 网络助手需求列表

1. 按规则代理本机出站流量，支持 `TCP`、`UDP`、`HTTP`，流量可直连或经探针出口。
2. 探针出口可为直接出口，也可为由多个探针级联形成的出口；与探针连接及级联链路均可为正向或反向。
3. 直连通过 `direct_whitelist` 定义。
4. 规则支持 `CIDR`、`IP`、`DOMAIN` 后缀，并支持按代理组配置。
5. 代理组为一组探针出口，由用户在界面中选择其一。
6. 本地与远方探针之间使用私有协议，承载链路可选 `TCP`、`HTTPS`、`HTTP3`、`WebSocket` 其一。
7. 网络助手支持链路管理：可新建、删除、修改探针出口，可设置链路协议，且级联链路各跳可使用不同链路协议。
8. `HTTPS` 提供 `codex` 鉴权接口，鉴权成功后按业务需求进入私有协议模式，鉴权失败后延时返回 `codex` 失败响应。
9. `Windows` 采用 `TUN` 网卡，`Linux` 使用 `TUN`。
10. 兜底规则在界面可选 `直连`、`代理组`、`拒绝`。
11. 直连链路或级联链路在拓扑设置完成后，管理端与探针自动维持链路，业务层复用链路连接，避免反复建立连接。
12. 探针服务接口通过网络助手链路管理进行设置；需支持配置探针服务监听地址/端口，并考虑 `NAT` 场景下服务端口与公网端口通过手动设置不一致的情况。
